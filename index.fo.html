ger eina nýggja gitgoymslu

ger ein nýggja mappu, lat hana upp og koyr
git init
fyri at gera eina nýggja gitgoymslu.

heinta eitt arbeiðsavrit av eini gitgoymslu

ger eitt arbeiðsavrit av einari lokalari gitgoymslu við at koyra
git clone /stíggjur/til/gitgoymslu
tá fjarambætari verður brúktur, koyr
git clone brúkaranavn@ambætaranavn:/stíggjur/til/gitgoymslu

arbeiðsgongd

tín lokala gitgoymsla er trý "trø", sum git umsitur. tað fyrsta er tín (Arbeiðs faldari) har tínar fílur liggja. tað næsta er Index sum er eitt pallsetingar øki og til endans er tað HEAD sum peikar á seinasta 'commit' sum tú hevur gjørt.

add & commit

Tú kann pallseta broytingar (leggja afturat Index) við at skriva
git add <fílunavn>
git add *
Hetta er fyrsta stig í vanligu git arbeiðsgongdini. Fyri at 'commit'-að pallsettu broytingarnar skrivar tú
git commit -m "Viðmerking um commit"
Nú eru fílurnar 'commit'-aðar til HEAD, men enn ikki í tíni fjargitgoymslu.

push - trýst broytingar út til (fjar)gitgoymslu

Broytingarnar hjá tær eru nú í HEAD á tínum lokala arbeiðsavriti. Fyri at senda hesar broytingarnar til tína (fjar)gitgoymslu, koyr so
git push origin master
Broyt master til einhvørja grein, sum tú vil trýsta tínar broytingar út til.

Um tú ikki hevur gjørt eitt avrit av einari verandi gitgoymslu og tú vil knýta tína gitgoymslu í eina (fjar)gitgoymslu, so kann hetta gerast við
git remote add origin <ambætari>
Nú ber so til at trýsta broytingarnar til tað valdu (fjar)gitgoymsluna

greinar

Greinar verða nýttar til menna táttir avbyegdir hvør frá øðrumn. master greinin er forsetta greinin, tá tú ger eina nýggja gitgoymslu. Brúka aðrar greinar til menning og 'mergja' tær aftur á master greinina tá tú er liðug/ur.

ger eina nýggja grein nevnda "feature_x" og skift til hana við at skriva
git checkout -b feature_x
skift aftur til master
git checkout master
and delete the branch again
git branch -d feature_x
a branch is not available to others unless you push the branch to your remote repository
git push origin <branch>

update & merge

to update your local repository to the newest commit, execute
git pull
in your working directory to fetch and merge remote changes.
to merge another branch into your active branch (e.g. master), use
git merge <branch>
in both cases git tries to auto-merge changes. Unfortunately, this is not always possible and results in conflicts. You are responsible to merge those conflicts manually by editing the files shown by git. After changing, you need to mark them as merged with
git add <filename>
before merging changes, you can also preview them by using
git diff <source_branch> <target_branch>

log

in its simplest form, you can study repository history using.. git log
You can add a lot of parameters to make the log look like what you want. To see only the commits of a certain author:
git log --author=bob
To see a very compressed log where each commit is one line:
git log --pretty=oneline
Or maybe you want to see an ASCII art tree of all the branches, decorated with the names of tags and branches:
git log --graph --oneline --decorate --all
See only which files have changed:
git log --name-status
These are just a few of the possible parameters you can use. For more, see git log --help

replace local changes

In case you did something wrong, which for sure never happens ;), you can replace local changes using the command
git checkout -- <filename>
this replaces the changes in your working tree with the last content in HEAD. Changes already added to the index, as well as new files, will be kept.

If you instead want to drop all your local changes and commits, fetch the latest history from the server and point your local master branch at it like this
git fetch origin
git reset --hard origin/master
